通过真机平台使用苹果设备连接测试环境网络，发现无法进行抓包，但是连接正式环境网络可以正常抓包，抓包工具使用的是 [whistle](https://github.com/avwo/whistle)。经过观察发现设备在测试环境网络状态栏不显示 Wi-Fi 图标，在正式环境则显示 Wi-Fi 图标。除了设备连接的 Wi-Fi 网络不同，即测试环境只能连接内网（无互联网连接），正式环境连接互联网，其余配置都相同。


### 模拟网络环境实验

- 苹果设备连接局域网（路由器没有接入外网）情况下，系统状态栏显示 WiFi 图标，并且支持抓包；
- 路由器处于无线桥接，苹果设备系统栏显示 WiFi 图标，并且支持抓包；
- 路由器开启 AP 隔离模式，代理设备通过无线连接路由器，苹果设备系统栏显示 WiFi 图标，不支持抓包；
- 路由器开启 AP 隔离模式，代理设备通过有线连接路由器，苹果设备系统栏显示 WiFi 图标，支持抓包；

> 开启 AP 隔离后，连接在 2.4G 无线网络下的设备之间将不能互相访问，且连接在 5G 无线网络下的设备之间也不能互相访问。但是连接在 2.4G 无线网络下的设备可以访问连接在 5G 无线网络下的设备。同时，连接在 2.4G 无线网络下的设备和连接在 5G 无线网络下的设备依然都可以访问有线连接设备。


### 发现问题

经过一系列实验后，发现连接测试网络的设备自动获取 DNS 失败（不知道运维对测试网络做了哪些配置和限制），导致现象就是在设置中显示已经连接 Wi-Fi，但是状态栏没有 Wi-Fi 图标。


### 解决方案

在设备连接测试网络下，需要在 Wi-Fi 设置中手动配置 DNS，添加的 DNS 服务器地址是该 Wi-Fi 的路由器地址。
