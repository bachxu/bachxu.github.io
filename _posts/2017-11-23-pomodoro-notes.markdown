在工作生活中，你有没有遇到过以下情景：明明在做着 A 这件事，结果做着做着切换到去做 B 了；或者前一秒还在全力做 A，后一秒思绪就跑偏，跑远了；亦或是本来打算聚精会神处理 A 这件事，结果不是自己想起来喝杯水，吃点东西，运动运动，就是突然手机收到信息，或者有人来找你谈事情等情况，结果到头来感觉自己什么事情都还没有做完。

番茄工作法的焦点并不在于时间，而是在如何用最简单的工具和方法来提高你的工作效率。

### 工具与方法

番茄工作法的流程由五个阶段构成：

| 事件 | 时间 | 内容 |
| :--:| :--: | :--: |
| 计划 | 一天的开始 | 计划一天要做的事 |
| 追踪 | 全天时间 | 记录好这一天内做过的工作学习等所有事情，作为原始数据，包括工作时间和其它感兴趣的指标 |
| 记录 | 一天的结束 | 建立一个日工作的档案，收集原始数据 |
| 分析 | 一天的结束 | 由原始数据分析出有用的信息（有用信息即能帮助你提高效率的信息） |
| 可视化处理 | 一天的结束 | 把有用的信息用鲜明的方式表示出来，以便清楚得看到自己的进步 |

番茄工作法所需要的工具有以下几样：

- 一个番茄计时器，可以是机械闹钟也可以是计时软件，比如 macOS 上的 Tomato One；
- 一张 TO DO TODAY 工作计划表，每天开始时填写以下事项：
    - 小标题，包括时间、地点、人物。
    - 清单，包含今天要做的事情，并按先做和后做的顺序排列。
    - 表格中一部分留给“计划外的紧急事件”。这些事件是不可预期的，但又必须要处理的，当它们出现在表格中时，可能会推翻全天的计划。
- 一份 ACTIVITY INVENTORY 活动清单，由以下两项组成:
    - 标题:人物、事件。
    - 想到什么要做的就记什么，随出现随记。一天的结束时，把完成的事件划去。
- 一份记录表，用于记录原始数据，写报告或制作图表时要用到。

### 游戏规则

- 一个番茄时间共30分钟，包括25分钟的工作时间和5分钟的休息时间。
- 每四个番茄时间过后，停止你的工作，进行一次较长时间的休息，大约15到30分钟。
- 一个番茄时间是不可分割的，不存在半个番茄时间或一刻钟的番茄时间。
- “保护你的番茄”，明确地告知想要打断你的人，并商量推迟他的请求。
- 如果一项任务的估测值大于5到7个番茄，那么就打散它。复杂的工作应该被分割为小的事情。
- 如果一项任务的估测值小于一个番茄，就把几个小任务组合成一个大任务。
- 内部打断如果可以推迟，则在正在进行的任务旁边标上点号（`），并在 ACTIVITY INVENTORY 活动清单中写下这件事，标上“U”，同时可以写上截止日期，然后继续工作。
- 外部打断，每当有人要打断你时，就在你的 TO DO TODAY 工作计划表中对应位置画上（-），采用“发现事件，记录，然后把事件放进之后的番茄时间内处理，或留到其它时间。”然后参照以下的做法：
    1. 如果你认为这项打断时重要的，那么就在 TO DO TODAY 工作计划表中的“计划外的紧急事件”一栏写下这件事。
    2. 同时在 ACTIVITY INVENTORY 活动清单中写下这件事，并标记“U“，如果需要的话标上截止日期。
- 每完成一个番茄时，在对应任务栏中画一个“x”。
- 每作废一个番茄时，在对应任务栏中画一个“。”。

TO DO TODAY 工作计划表：用于计划与分配每天需要做的事情和评估所需的番茄数，并留有计划外的紧急事件列表空间，用于处理不可预期又必须处理的事件。
