[JMeter](https://jmeter.apache.org/) 最早的用途是为了测试 Tomcat 的前身 JServ 的执行效率。JMeter 是一款使用 Java 开发的、开源免费的测试工具，主要用来做 API 的功能测试和性能测试（压力测试/负载测试）。

### JMeter 简介

JMeter 可以用来测试静态或者动态资源的性能。最初 JMeter 是为 Web/HTTP 测试而设计的，但是随着发展，它的功能也得了不断扩展。下面罗列了 JMeter 的一些主要功能。

- 能对 HTTP 和 FTP 服务器进行压力与性能测试；
- 允许通过多个线程并发取样和通过单独的线程组对不同的功能同时取样；
- 精心设计的 GUI 允许用户快速操作和精确计时；
- 可以缓存和离线分析/回放测试结果；
- 可链接的取样器允许无限制的测试能力；
- 有各种负载统计表和可链接的计时器；
- 数据分析和可视化插件提供了很好的可拓展性；
- 提供动态输入功能；
- 支持脚本变成的取样器；

### JMeter 的主要概念

- 测试计划：是使用 JMeter 进行测试的起点，是其他 JMeter 测试元件的容器；
- 线程组：代表一定数量的并发用户，可以用来模拟并发用户发送请求。实际的请求内容在取样器中定义，线程组包含请求内容；
- 监听器：负责收集测试结果，同事控制结果的显示方式；
- 逻辑控制器：可以自定义 JMeter 发送请求的行为逻辑，同时使用它与取样器可以模拟复杂的请求序列；
- 断言：用来判断请求响应的结果是否与期望一致；
- 配置元件：维护取样器需要的配置信息，并根据实际的需要会修改请求的内容；
- 前置处理器和后置处理器：负责在生成请求与之后完成工作；
- 定时器：负责定义请求之间的延迟间隔控制；

### JMeter 的简单使用

JMeter 的安装非常简单，在安装完 Java 环境的基础上，从 JMeter 官网上下载 JMeter，解压之后即可使用。以 macOS 为例，进入到解压文件的 bin 目录下，使用如下启动命令即可启动。

```shell
$ ./jmeter
```

JMeter 的使用步骤一般如下。

1. 观察被测对象，了解被测对象的功能以及表现；
2. 新建 JMeter 的线程组；
3. 创建一个 HTTP 请求；
4. 添加 HTTP Header Manager；
5. 添加 View Results Tree，用来查看运行结果；
6. 运行测试，查看结果；
7. 添加 Response Assertion 和 Assert Results；
8. 添加 User Defined Variables；
9. 添加关联；
10. 并发测试；
