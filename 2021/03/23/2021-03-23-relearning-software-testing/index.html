重新学习软件测试

Posted on 2021-03-23 Edited on 2023-05-26 In iTechnology

记得第一次接触软件测试是在大学课堂上，通过 QTP 软件进行简单录制和点击网页和客户端上的一些功能。随着工作几年之后，现在回过头重新学习软件测试方面的知识，发现软件测试的世界也是如此精彩，不仅要有扎实的理论基础，还需要具备行业背景知识，对整个产品链路知根知底，有清晰的逻辑思维以及足够完善的技术栈。

最近阅读了两本关于软件测试方面的书：一本是《软件测试的艺术》，主要介绍了软件测试方面的理论知识；另一本是《测试工程师全栈技术进阶与实践》，看书名就知道是更偏向实践层面。通过这两本书的帮助，我对软件测试有了一个更深刻的认知。

软件测试的心理学和经济学

软件测试的心理学

测试是为了发现错误而执行程序的过程。只有正确的认识软件测试，才能在工作中不去排斥，更有成就地去完成它。

软件测试的经济学

我们不能够发现“所有”的错误，为了应对测试经济学的挑战，需要在开始测试之前建立某些策略。而黑盒测试和白盒测试是两种最普遍的策略。

测试指导原则

  1. 测试用例中一个必需部分是对预期输出或结果进行定义；
  2. 程序员应该避免测试自己编写的程序；
  3. 编写软件的组织不应当测试自己编写的软件；
  4. 应当彻底检查每个测试的执行结果；
  5. 测试用例的编写不仅应当根据有效和预料到的输入情况，而且也应当根据无效和未预料到的输入情况；
  6. 检查程序是否“未做其应该做的”仅是测试的一半，测试的另一半是检查程序是否“做了其不应该做的”；
  7. 应避免测试用例用后即弃，除非软件本身就是一个一次性的软件；
  8. 计划测试工作时不应默许假定不会发现错误；
  9. 程序某部分存在更多错误的可能性，与该部分已发现错误的数量成正比；
  10. 软件测试是一项极富创造性、极具智力挑战性的工作；

测试用例的设计

常用测试用例的设计方法主要分为白盒测试和黑盒测试。

黑盒测试，又称数据驱动测试，主要包含以下方法：

  • 等价类划分
  • 边界值分析
  • 因果图分析
  • 错误猜测

白盒测试，又称逻辑驱动测试，主要包含以下方法：

  • 语句覆盖
  • 判定覆盖
  • 条件覆盖
  • 判定/条件覆盖
  • 多重条件覆盖

一个“好的”测试用例应当具备以下 3 个特征。

  • 整体完备性：“好的”测试用例一定是一个完备的整体，是有效测试用例组成的集合，能够完全覆盖测试需求。
  • 等价类划分的准确性：指的是对于每个等价类都能保证只要其中一个输入测试通过，其他输入也一定测试通过。
  • 等价类集合的完备性：需要保证所有可能的边界值和边界条件都已经正确识别。

功能测试

功能测试是一个试图发现程序与其外部规格说明之间存在不一致的过程。功能测试通常是一项黑盒操作。

系统测试（非功能测试）

系统测试并非是测试整个系统或程序功能的过程。系统测试有着特定的目的，将系统或程序与其初始目标进行比较。系统测试可以用到的测试用例：

  1. 能力测试：确保程序的目标功能实现
  2. 容量测试：发现处理大容量数据时的程序异常
  3. 强度测试：发现在大规模负载、高强度不间断持续的数据处理中的异常
  4. 可用性测试：评估最终用户在使用软件并与软件交互时的可用性问题
  5. 安全性测试：试图攻破程序的安全防线
  6. 性能测试：评估程序的响应时间以及吞吐量瓶颈
  7. 存储测试：确保程序可以正确处理其对存储的需求，包括系统的存储和物理上的存储
  8. 配置测试：检查程序是否能在推荐配置上流畅运行
  9. 兼容性/转换测试：评估新版本是否能兼容老的版本
  10. 安装测试：确保能够在所有支持的平台上安装软件
  11. 可靠性测试：评估程序是否能达到规格说明中的运行时长和 MTBF 要求
  12. 可恢复性测试：测试系统恢复相关的功能是否按设计要求实现
  13. 服务/可维护性测试：评估系统是否拥有良好的数据处理和日志机制，以备技术支持和调试之需
  14. 文档测试：校验所有的用户文档是否准确
  15. 过程测试：对软件系统操作或维护所需设计的流程进行评估和确定

高效撰写软件缺陷报告

缺陷报告是测试工程师与开发工程师沟通的重要桥梁，也是测试工程师日常工作的重要“输出”。企业都有缺陷管理系统，如 JIRA、BugZilla、BugFree 和 Tapd 等。当使用这类系统提交缺陷时，会自动生成报告模版，需要注意的是，我们应该提供准确有用的信息。一份高质量的缺陷报告主要有以下部分组成：

  • 缺陷标题
  • 缺陷概述
  • 缺陷影响
  • 环境配置
  • 前置条件
  • 缺陷重现步骤
  • 期望结果和实际结果
  • 优先级和严重程度
  • 变通方案
  • 根原因分析
  • 附件

常用工具

  • Selenium with Python：GUI 自动化测试。
  • Appium：移动应用测试工具。
  • Requests：Requests 是一个优雅而简单的 Python 的 HTTP 库，为人类构建。
  • Postman：Http 请求模拟工具，用于 API 测试。
  • Charles：抓包工具。
